git-introduce

介绍

Git是分布式版本控制系统。每个人电脑上都有一个完整的版本库，即使没有网络，也能实现版本控制。在实际的使用中，通常有一台充当“中央服务器”的电脑用于方便成员交换”修改”。

安装

在Linux上安装Git: sudo apt-get install git
在Windows上安装Git：下载安装程序，然后在工作目录右键，“Git Bash Here”
安装完以后配置一下用户名和邮箱（若需对某个仓库指定不同的用户名和密码，去掉--global参数，在该仓库下运行相同命令）：

创建版本库

在一个空目录下运行git init，git就会将该目录初始化为git仓库，仓库建好以后，该目录中所有文件变动都将被git记录。这时你会发现原目录下多了一个.git目录，它就是用来存放版本信息的，轻易不要改动其中的内容。
另外需要说明的是，git只能跟踪文本文件的改动，对于二进制文件，它们能被git管理，但无法跟踪文件的变化。Microsoft的Word格式是二进制的。
还有，不要用Windows的记事本编辑文本文件，因为记事本会给每个文件开头加个0xefbbbf的字符，这可能导致原本正确的程序无法通过编译。

使用版本库

首先在git目录下编写一个文本文件readme.txt，内容自定。
把一个文件放到git仓库只需两步：

  1. 用git add告诉git把文件添加到仓库：

git commit -m “wrote a readme file”

ssh-keygen -t rsa -C “myemail@email.com“

git remote add origin

git branch dev
git checkout dev

git stash apply stash@{0}
git stash drop stash@{0}

标签

标签相当于是commit_id的别名
git tag查看所有标签
git tag <name> <commit_id>为commit_id打上名为name的标签，若commit_id为空，则默认打在最新提交的commit_id上。
git tag -a <name> -m <tagmessage>
git show <tagname>查看标签信息

查看某个文件为何被.gitignore忽略

git check-ignore -v <file>

配置命令别名

git config --global alias.st status
git config --global alias.co checkout
git config --global alias.unstage 'reset HEAD'
git config --global alias.lg "log --color --graph --pretty=format:'%Cred%h%Creset -%C(yellow)%d%Creset %s %Cgreen(%cr) %C(bold blue)<%an>%Creset' --abbrev-commit"
